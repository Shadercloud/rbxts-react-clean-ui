This guide explains how to set up a local environment for developing **React Clean UI** or contributing to the documentation.

<Callout type="info">
This guide assumes you're already familiar with **roblox-ts**, **Rojo**, and **Roblox Studio**.
</Callout>

# Developing React Clean UI

## 1. Create a development project

Create a new roblox-ts project outside of the React Clean UI repository.

```bash
mkdir C:\Users\username\Documents\dev-game
cd C:\Users\username\Documents\dev-game

npm init roblox-ts game
```

When prompted:

- Leave **Project directory** blank.
- Answer **Y** to the remaining questions.

## 2. Install the required packages

Install the packages required for the development project.

```bash
npm install @rbxts/react @rbxts/react-roblox @rbxts/ui-labs @rbxts/services
```

## 3. Clone React Clean UI

Create a `/src/Packages` directory and clone the repository into it.

```bash
cd src
mkdir Packages
cd Packages

git clone https://github.com/Shadercloud/rbxts-react-clean-ui.git
```

Your project should now look like this:

```text
dev-game/
├── src/
│   └── Packages/
│       └── rbxts-react-clean-ui/
```

## 4. Configure TypeScript

Replace your root project's `tsconfig.json` with the following:

```json
{
	"compilerOptions": {
		"allowSyntheticDefaultImports": true,
		"downlevelIteration": true,
		"jsx": "react",
		"jsxFactory": "React.createElement",
		"jsxFragmentFactory": "React.Fragment",
		"module": "commonjs",
		"moduleResolution": "Node",
		"noLib": true,
		"resolveJsonModule": true,
		"experimentalDecorators": true,
		"forceConsistentCasingInFileNames": true,
		"moduleDetection": "force",
		"strict": true,
		"target": "ESNext",
		"typeRoots": [
			"node_modules/@rbxts"
		],
		"types": [
			"types"
		],
		"rootDir": "src",
		"outDir": "out",
		"baseUrl": "src",
		"incremental": true,
		"tsBuildInfoFile": "out/tsconfig.tsbuildinfo",
		"paths": {
			"@rbxts/react": [
				"../node_modules/@rbxts/react"
			],
			"@rbxts/react-roblox": [
				"../node_modules/@rbxts/react-roblox"
			],
			"@rbxts/ui-labs": [
				"../node_modules/@rbxts/ui-labs"
			],
			"@rbxts/services": [
				"../node_modules/@rbxts/services"
			],
			"@rbxts/react-clean-ui": [
				"Packages/rbxts-react-clean-ui"
			]
		}
	},
	"exclude": [
		"src/Packages/**/src",
		"src/Packages/**/node_modules",
		"src/Packages/**/out",
		"src/Packages/**/.vscode",
		"src/Packages/**/docs"
	]
}
```

## 5. Configure Rojo

Create a `default.project.json` file in the root of your development project.

```json
{

	"globIgnorePaths": [

		"**/tsconfig.json"

		"ServerScriptService": {

			"TS": {
				"$path": "out/server"
			}
		},
		"ReplicatedStorage": {
			"$className": "ReplicatedStorage",
			"rbxts_include": {
				"$path": "include",
				"node_modules": {
					"$className": "Folder",
					"@rbxts": {
						"$className": "Folder",
						"compiler-types": {
							"$path": "node_modules/@rbxts/compiler-types"
```

## 6. Start the development environment

You'll need **three terminals** running simultaneously.

### Terminal 1 — Compile your game

```bash
cd C:\Users\username\Documents\dev-game
npx rbxtsc -w
```

### Terminal 2 — Compile React Clean UI

```bash
cd C:\Users\username\Documents\dev-game\src\Packages\rbxts-react-clean-ui
npx rbxtsc -w
```

### Terminal 3 — Run Rojo

```bash
cd C:\Users\username\Documents\dev-game
rojo serve
```

## 7. Open Roblox Studio

Open your development place in Roblox Studio and connect it to the running Rojo server.

## 8. Install UI Labs

Install the UI Labs plugin:

https://ui-labs.luau.page/docs/installation

Once installed, open **Plugins → UI Labs**. You should see the React Clean UI component stories, allowing you to preview components while you develop.

# Writing the Documentation

The documentation is built using **Fumadocs**.

## 1. Clone the repository

```bash
git clone https://github.com/Shadercloud/rbxts-react-clean-ui.git
```

## 2. Install dependencies

```bash
cd rbxts-react-clean-ui/docs
npm install
```

## 3. Start the development server

```bash
npm run dev
```

## 4. Open the documentation

Open the following URL in your browser:

```text
http://localhost:3000
```

## 5. Start editing

Documentation pages are located in:

```text
docs/content/docs/
```

Any changes you make will automatically reload in your browser while the development server is running.
